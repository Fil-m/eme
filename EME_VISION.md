# EME OS: Vision & Architecture Manifesto

## 1. Головна Візія (The Vision)
**EME OS** — це не просто застосунок, це **персональна, автономна операційна система для виживання, співпраці та управління життям**. 
Система створюється як офлайн-first пісочниця з підтримкою децентралізованих Mesh-мереж, інтегрованим локальним ШІ та абсолютною власністю на дані (Data Sovereignty). ЕМЕ покриває всі аспекти життя: від ведення проектів (Канбан) до комунікації (Соцмережа), навчання (База Знань) та виживання.

## 2. Шлях користувача (User Journey)
1. **Інсталяція:** Користувач завантажує "Ядро" (Core) - легке Python-середовище.
2. **Onboarding (Майстер налаштування):** При першому запуску система ініціалізує власний вузол (Node), створює локальний профіль.
3. **Вибір Пресетів:** Користувачу пропонується обрати стартовий набір (наприклад: "Для родини", "Соціальна мережа", "Керування проектами", "Ігрова майстерня", "Виживання").
4. **Використання:** Система працює повністю локально (офлайн). Якщо поруч з'являються інші пристрої з EME (через Wi-Fi/Bluetooth) — відбувається Mesh-синхронізація. ШІ-агенти (Ollama) працюють локально, допомагаючи структурувати дані.
5. **Розширення:** Користувач може кастомізувати екран, змінювати положення модулів, завантажувати нові з магазину або від інших розробників.

## 3. Модульна Архітектура (Core vs Modules)

Система розділена на **Ядро (Core)** та **Модулі розширення (Apps/Plugins)**.

### EME Core (Ядро)
*Відповідає виключно за базове виживання системи.*
- **Identity & Auth:** Профілі, криптографія, сесії.
- **Mesh Engine:** Протокол синхронізації пристроїв (UDP/TCP виявлення, конфлікт-менеджмент).
- **UI Shell:** Головний екран, рендерер компонентів (Vue3 SFC Loader), глобальний стор налаштувань.
- **AI Router:** Інтерфейс до локальних моделей (LLM).

### Modules (Додатки)
*Повністю ізольовані сутності.*
- **Модульність інтерфейсу:** Кожен модуль має свої `.vue` файли. Фронтенд EME завантажує їх динамічно в рантаймі (Runtime).
- **Ізоляція Даних (Database-per-module):** Кожен модуль може мати власну базу даних (`module_name.sqlite3`). Це дозволяє легко бекапити, шерити або видаляти модуль без ризику пошкодити ядро або інші модулі.
- **Сторонні модулі (Third-Party Plug&Play):** Оскільки архітектура базується на стандартних API та динамічному Vue, будь-який розробник з GitHub може створити папку з модулем. Ядро автоматично підхоплює його, додає в меню та надає доступ до Core API.

## 4. Плюси та мінуси ізольованих баз даних (Module DBs)

**Плюси:**
- Краш чи корупція даних одного модуля не ламає всю EME OS.
- Можливість ділитися окремими базами в Mesh (наприклад, розшарити лише базу "Енциклопедія виживання", але не базу "Особисті нотатки").
- Високий рівень приватності: різні бази можна шифрувати різними ключами.

**Мінуси:**
- Ускладнений крос-модульний пошук (запит має іти в кілька баз одночасно).
- Ядро повинно виступати "оркестратором" між базами (модуль може посилатися на `user_id`, але дані користувача лежать у Core-базі).

## 5. Дорожня карта (Next Steps)
1. **Рефакторинг Ядра:** Відокремити жорсткі прив'язки поточних модулів від `eme/settings.py` в динамічний "Менеджер Плагінів".
2. **Setup Wizard:** Створити процес ініціалізації при першому вході без хардкоду.
3. **Database Routing:** Налаштувати в Django систему `DATABASE_ROUTERS`, щоб `eme_kb` записувала дані в `kb.sqlite3`, а `eme_chat` в `chat.sqlite3`.
4. **App Store Concept:** Додати можливість скачувати модулі (папки з `.py` та `.vue`) в рантаймі і підключати їх "на гарячу".

## 6. Ідеї для майбутньої інтеграції (Catalog Backlog)
Ось перелік потужних Open Source модулів для майбутнього розширення (відкладені на потім):
- **Trilium Notes**, **SiYuan**, **GPT Researcher** (Бажана База Знань та автономні ШІ-дослідження).
- **Taiga**, **Leantime**, **4ga Boards** (Альтернативи для командної роботи).
- **Vaultwarden** (Локальний менеджер паролів — надзвичайно бажана інтеграція).
- **Immich**, **PhotoPrism**, **Jellyfin**, **Audiobookshelf** (Медіасервери та ШІ-галереї, запуск через Docker).
- **Gitea/Forgejo**, **Appsmith**, **Coolify**, **Paperless-ngx** (DevOps, Low-code та розпізнавання паперів).
- **Pi-hole / AdGuard Home**, **Proxmox VE** (Системна інфраструктура будинку).
